# 技术路线与实现地图

## 总体拆法

数字人导游不需要一次做成复杂实时系统，可以拆成五个模块：

1. 形象：苏丽娘 2D 立绘和角色设定。
2. 驱动：图生视频、对口型或轻动效，让她能说话。
3. 声音：中 / 英 / 意 TTS 或真人配音。
4. 大脑：大模型 + 苏州知识库，回答游客问题。
5. 互动壳：网页或小程序，承载景点选择、问答、合影、路线定制。

比赛 Demo 的策略是「预渲染视频 + 可点交互」结合：关键讲解用精修视频保证观感，问答和路线用网页证明实时能力。

## 当前技术实现

- 前端：单页 `index.html`，原生 JavaScript，无构建流程。
- 后端：Python 标准库 `http.server`，端口 8770。
- 内容：`data/content.json` 集中管理人设、欢迎词、景点、路线、知识库、FAQ。
- 问答：DeepSeek API。
- 路线生成：DeepSeek JSON 模式。
- 合影：UU 中转 gpt-image-2。
- 兜底：问答、路线、合影都有失败回退。

## 推荐技术路线

### 形象驱动

优先测试：

- 即梦对口型。
- 可灵对口型。
- Hedra talking photo。

测试方式：同一张苏丽娘主立绘、同一段 10 秒中文配音，横向比较口型、脸部漂移、头饰稳定度和输出清晰度。

如果都不稳定，退路是：

- 半身轻动效：眨眼、点头、水袖微动、字幕同步。
- Live2D：效果更好，但时间和学习成本更高。

### 声音

主线做中 / 英 / 意三语，不硬做苏州话 TTS。苏州话可以作为真人录制的一句评弹腔彩蛋。

推荐优先级：

1. 中文普通话女声，温婉慢速。
2. 英文自然导览腔。
3. 意大利语清晰、标准，作为差异化记忆点。
4. 苏州话 / 评弹腔一句彩蛋。

### 大脑与知识库

短期：继续用 DeepSeek + 精心写好的知识上下文。

中期：用 Coze 或 Dify 搭「苏丽娘」智能体，上传苏州知识库，包括：

- 拙政园景点。
- 昆曲与《牡丹亭》。
- 苏绣、评弹、苏州园林。
- 平江路、山塘街、苏州博物馆。
- 常见游客问题。

### 互动壳

当前网页 Demo 足够比赛演示。后续如果要产品化，可迁移到：

- 微信小程序。
- 景区扫码 H5。
- 文博会大屏互动页。
- 导览屏 / 触摸屏。

## 技术风险

| 风险 | 判断 | 对策 |
|---|---|---|
| 卡通立绘对口型崩脸 | 最大风险 | 先做 10 秒横评，不满意就改轻动效 |
| 苏州话 TTS 不成熟 | 高概率 | 不做主线，只做真人彩蛋 |
| 网络不稳定 | 现场常见 | 保留所有兜底，不依赖实时接口完成主流程 |
| 实时定位太复杂 | 不适合当前周期 | 用手动选景点或二维码触发替代 |
| 多语内容维护成本高 | 中等 | 所有新增文案必须同步 zh / en / it |

## 开发约定

- 不提交 `.env` 或任何密钥。
- 内容优先改数据文件，不把文案写死进页面。
- 新增景点必须补齐三语名称和讲解。
- 新增接口必须有失败兜底。
- 演示优先保证「点了有反馈」，不要让用户看到空白或报错。
