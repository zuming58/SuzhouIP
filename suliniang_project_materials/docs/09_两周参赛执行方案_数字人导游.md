# 两周参赛执行方案：苏丽娘数字人导游

## 结论先行

不要把目标定成「完整实时 3D 数字人」。两周内最稳、最像参赛作品、也最能打动评委的形态是：

> 一个手机端优先的 AI 数字人导游 Demo：以苏丽娘 3D 形象为主视觉，提供欢迎视频、语音/文字对话、多语言讲解、情绪化反馈、路线定制、一个景区的沉浸式讲解闭环，并用预渲染动作视频 + 实时大模型问答结合，证明它未来可以升级为真正实时数字人。

核心策略是「实时大脑 + 预渲染身体」：

- 大脑实时：用户语音/文字提问，模型实时回答。
- 声音实时：TTS 把回答朗读出来。
- 身体半实时：根据状态切换苏丽娘的动作图或短视频，不追求每一句实时精准口型。
- 重点讲解预渲染：欢迎、景点讲解等核心场景用提前生成好的视频，保证观感。

## 为什么不建议两周内硬做完整实时 3D 数字人

完整实时 3D 数字人至少要同时解决：

1. 3D 模型建模或可驱动 Live2D/3D 角色。
2. 实时口型绑定。
3. 实时表情和动作状态机。
4. 语音识别、LLM、TTS 串联。
5. 手机端性能和延迟。
6. 稳定 Demo 和比赛包装。

这条路风险太集中。当前我们已经有高质量 3D 图片资产，但还没有可骨骼驱动的 3D 模型文件。把图片变成可实时驱动模型，是另一个项目，不适合在最后两周押宝。

## 推荐作品形态

### 名称

苏丽娘 AI 数字人导游

### 一句话

从昆曲《牡丹亭》中走出的苏州文化精灵，能听懂游客问题、用中英意多语言讲解苏州，并根据情绪和场景切换动作，为游客定制路线和沉浸式导览。

### Demo 主流程

1. 打开手机页面。
2. 苏丽娘播放一段 10-20 秒欢迎视频，可跳过。
3. 进入导游首页，显示苏丽娘 3D 形象和几个入口：
   - 问苏丽娘
   - 定制苏州路线
   - 拙政园讲解
   - 多语言切换
4. 用户语音或文字提问。
5. 系统识别语音，调用大模型生成回答。
6. 系统用 TTS 朗读回答，同时苏丽娘切换到「张嘴 / 讲解」状态。
7. 如果用户要路线，苏丽娘切换到「托腮思考」，生成路线后切换「颔首邀请」或「抬手指景」。
8. 进入拙政园 Demo，点击一个景点，播放预制讲解视频或音频 + 动作图。
9. 支持中文、英文、意大利语；苏州话可以做一句彩蛋问候。

## 必须做出的功能

| 模块 | 比赛要求对应 | 两周内实现方式 |
|---|---|---|
| 智能对话 | 大模型、智能服务 | DeepSeek / 豆包 / Kimi 等模型接口，支持文字和语音输入 |
| 情感交互 | 情绪反馈、拟人感 | 用用户意图和关键词判断状态，切换不同动作图、语气和反馈文案 |
| 多语言服务 | 国际化导览 | 中文 / 英文 / 意大利语三语切换；至少核心讲解三语 |
| 语音合成 | 朗读讲解 | TTS 朗读模型回答；可先做普通话，三语按优先级补 |
| 动作驱动 | 数字人动作 | 不做真骨骼驱动，改成状态驱动：听、想、说、指景、笑、欢迎 |
| 路线定制 | 个性化服务 | 用户输入天数、同行人、偏好，模型生成路线 |
| 沉浸讲解 | 历史遗迹讲解 | 选拙政园一个景点或五个景点，预制精品讲解 |

## 情感交互怎么做

情感交互不一定等于复杂情绪模型。比赛 Demo 里可以做成「可感知的情绪状态机」：

| 用户状态 / 意图 | 苏丽娘动作 | 话术风格 |
|---|---|---|
| 刚进入 | 颔首邀请 | 温柔欢迎 |
| 用户正在说话 | 侧耳倾听 | “我在听呢” |
| 系统生成答案 | 托腮思考 | “让我想一想” |
| 正在回答 | 张嘴 / 抬手指景 | 清晰讲解 |
| 用户问诗词/昆曲 | 闭眼 | 更慢、更有韵味 |
| 答对/彩蛋/合影 | 掩口轻笑 | 轻松亲切 |
| 等待下一步 | 整理水袖 | 安静待机 |

这样就是「动作 + 语气 + 场景」形成情感，而不是只在屏幕上显示一段文字。

## 动作驱动怎么做

短期不要做骨骼级动作驱动。我们现在有 3D 姿势图，最适合做「状态驱动」：

1. 每个状态绑定一张图。
2. 切换状态时加淡入淡出、轻微浮动、呼吸感。
3. 说话时使用 `expression_mouth_open.png`，配合音频播放和字幕。
4. 景点讲解时使用 `pose_point_to_scene.png`。
5. 生成路线时使用 `pose_thinking.png`。

如果时间允许，再做 3-5 个短视频：

- 欢迎视频。
- 指景讲解视频。
- 闭眼吟诵视频。
- 侧耳倾听动效。
- 水袖待机动效。

这些可以用图片转视频工具生成，作为“动作驱动”的可视化证据。

## 讲解内容取舍

不要贪多。主 Demo 建议做：

- 一个景区：拙政园。
- 五个点位：远香堂、与谁同坐轩、小飞虹、香洲、待霜亭。
- 一个重点点位做精品：与谁同坐轩或远香堂。

比赛演示时可以说：

> 当前 Demo 已完成拙政园样板，后续同样的方法可扩展到苏州博物馆、平江路、山塘街、虎丘等文旅场景。

## 推荐技术架构

```text
手机网页
  ├─ 苏丽娘动作状态层：图片/短视频切换
  ├─ 语音输入：浏览器语音识别或录音上传
  ├─ 大模型：回答问题、生成路线、翻译/多语言
  ├─ TTS：把回答变成语音
  ├─ 字幕：同步显示回答文本
  └─ 景点模块：预制讲解 + 地图/卡片
```

## 参赛交付物建议

1. 可点击手机网页 Demo。
2. 1 分钟作品宣传片。
3. 2-3 分钟演示录屏，展示语音问答、路线定制、多语言、景点讲解。
4. 方案 PPT：解释技术架构、四把锁、赛后转化。
5. 角色资产页：展示 3D 三视图、表情、动作、周边可能。

## 两周排期

### 第 1-2 天：定范围

- 明确只做手机网页 Demo。
- 确认模型接口和 TTS 方案。
- 整理拙政园知识库。
- 确认三语范围：至少中文全功能，英文/意大利语覆盖欢迎、景点讲解、核心问答。

### 第 3-5 天：做核心交互

- 语音/文字输入。
- 大模型回答。
- TTS 朗读。
- 苏丽娘状态切换。
- 字幕显示。

### 第 6-8 天：做路线定制和景点讲解

- 路线表单。
- 路线生成结果卡片。
- 拙政园点位卡片。
- 一个重点点位精品讲解。
- 五个点位基础讲解。

### 第 9-10 天：做视觉和动作

- 欢迎视频或欢迎动效。
- 状态图切换动画。
- 景点讲解短视频或音频。
- 移动端界面打磨。

### 第 11-12 天：包装参赛材料

- 录屏演示。
- 作品宣传片。
- PPT。
- 项目简介。

### 第 13-14 天：测试和兜底

- 断网/接口失败兜底。
- 手机端测试。
- 演示流程彩排。
- 压缩和提交材料。

## 最终判断

我们应该做的是：

> 一个“看起来像数字人、交互上是真智能、讲解上有沉浸感、技术上可解释、两周内能稳定交付”的 AI 虚拟导游样板。

评委不一定要求你已经实现商用级 3D 实时驱动，但会看你有没有把要求拆开并完成关键闭环：会听、会想、会说、会动、会用多语言服务游客、会给个性化路线。

苏丽娘目前最好的打法，就是用已经定版的 3D 角色资产做前台人格，用大模型和 TTS 做后台能力，用动作状态机模拟情感交互，用预制视频保证关键讲解效果。
